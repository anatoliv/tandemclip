# clipboardd

LAN-only clipboard sync for multiple Macs. No cloud, no relay, no remote
control — just plain-text clipboard content shared between machines you've
paired with a shared code. Runs as a menu-bar-only background agent.

**Status:** MVP scaffold (plain text only). See [Roadmap](#roadmap).

## Why this exists

Universal Clipboard requires the same Apple ID and Bluetooth proximity, and is
often disabled on managed machines. `clipboardd` needs neither — any Macs on the
same LAN that share a pairing code sync their clipboards, and nothing leaves the
LAN.

## How it works

```
 ┌── ClipboardWatcher ──┐        ┌── Transport ──────────────┐
 │ polls NSPasteboard   │        │ Bonjour: _clipboardd._tcp │
 │ changeCount (~0.4s)  │        │ PSK-TLS over Network.fw   │
 │ skips concealed types│        │ 4-byte len + JSON frames  │
 └──────────┬───────────┘        └─────────────┬─────────────┘
            │                                   │
            └──────────► SyncEngine ◄───────────┘
                     dedup (SHA-256 hash)
                     echo-loop prevention
                     last-writer-wins
                          │
                   MenuBarController
              (pause · peers · last sync · pairing code)
```

- **No push API for the clipboard** — macOS only exposes `changeCount`, so the
  watcher polls. It reads pasteboard *types* first and never pulls the data for
  content marked secret.
- **Password managers are respected.** Content tagged with the
  [nspasteboard.org](https://nspasteboard.org) markers `ConcealedType`,
  `TransientType`, or `AutoGeneratedType` is never synced.
- **Echo loops** are prevented two ways: applying a remote clip advances the
  watcher's baseline `changeCount`, and every clip is keyed by content hash so
  the same content never round-trips.

## Security model

- The **pairing code** is the only shared secret. It's stretched to a 256-bit
  key and used as a **PSK-TLS** pre-shared key, so every connection is
  authenticated *and* encrypted. Wrong code → TLS handshake fails → peer
  rejected. Being on the same Wi-Fi grants nothing on its own.
- LAN-only. No internet relay, no unauthenticated writes.
- Set the **same pairing code on every Mac.** Read/copy it from the menu on one
  machine and enter it on the others (currently via `defaults`, see below).

> ⚠️ MVP hardening TODO: replace SHA-256 code-stretch with HKDF, and move the
> pairing code out of `UserDefaults` into the Keychain.

## Build & run (development)

```sh
swift build
.build/debug/clipboardd        # a 📋 appears in the menu bar
```

Set a matching pairing code on each machine (until in-app entry lands):

```sh
defaults write net.amnesia.clipboardd pairingCode "K7QM-3PXF"
```

(The bundle/domain used by `UserDefaults` is the executable's identifier; when
run unbundled it falls back to the binary name. Copy the code shown in the menu
from your first Mac and set it on the others.)

## Install as a login agent

```sh
swift build -c release
sudo cp .build/release/clipboardd /usr/local/bin/clipboardd
cp LaunchAgent/net.amnesia.clipboardd.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/net.amnesia.clipboardd.plist
```

Logs: `/tmp/clipboardd.out.log`, `/tmp/clipboardd.err.log`.

### Managed / corporate Macs — check first

If these are MDM-managed machines, before relying on this:

1. Confirm you can run an unmanaged LaunchAgent (some profiles block it).
2. Confirm inbound LAN connections between the Macs are allowed by the host
   firewall / EDR (`nc -l` on one, `nc` from another on the same port).
3. For launch-at-login to survive Gatekeeper, sign with a Developer ID cert and
   notarize, or get the binary allow-listed.

## Roadmap

MVP is **plain text only** on purpose. Deferred until the text path is solid:

- [ ] In-app pairing-code entry + Keychain storage; HKDF key derivation
- [ ] Rich text (`public.rtf`)
- [ ] Images (`public.png`, `public.tiff`) with chunked framing
- [ ] File URLs (`public.file-url`)
- [ ] Per-device identity / allowlist (public-key pinning) beyond shared PSK
- [ ] Optional clipboard history (off by default)
